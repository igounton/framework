# AIDD flow curriculum

## Foundations (state-aware)

- Existing (code present): memory first, stack skipped. `project memory` → `connect to the AI`.
- Greenfield (empty): `design tech stack` → `project memory` → `connect to the AI`.

## Feature flow — 8 steps

| Step         | What                        | Why                            |
| ------------ | --------------------------- | ------------------------------ |
| brainstorm   | vague idea → precise        | fuzzy idea makes every step guess |
| spec*        | the contract of what to build | one agreed definition, no drift |
| plan         | break into phases           | small units that ship + verify |
| implement    | build phase by phase        | verified phases beat one leap  |
| assert       | check it behaves            | catch breakage before review   |
| review       | verdict before shipping     | ship or iterate, decided       |
| commit       | save the work               | atomic, reversible record      |
| pull request | ship it                     | open for merge                 |

`[map]`: `brainstorm → spec* → plan → implement → assert → review → commit → PR`

`*`: optional
