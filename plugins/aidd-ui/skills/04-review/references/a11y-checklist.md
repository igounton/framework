# Accessibility checklist

The a11y items the `02-accessibility` action walks against each screen.

- [ ] Contrast: text and UI meet WCAG AA against their background.
- [ ] Focus: every interactive element has a visible focus state.
- [ ] Keyboard: the whole flow is operable without a pointer.
- [ ] Semantics: elements use the right role; headings nest in order.
- [ ] Labels: inputs, icons, and controls have accessible names.
