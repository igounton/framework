# 01 - wireframe

Fixes the product's structure before any visual design: the information architecture (screens + navigation) and a low-fidelity sketch of each screen.

## When to use

- `PRODUCT.md` exists and the screens and their layout need to be settled before design.

## When NOT to use

- You want colors, type, or tokens → use [02-design-system](../02-design-system/README.md).
- You want to write UI code → use [03-build](../03-build/README.md).

## How to invoke

```
Use skill aidd-ui:01-wireframe
```

## Outputs

- A flow/IA map and a low-fidelity wireframe per screen — structure only.

## Prerequisites

- `PRODUCT.md` from [00-product](../00-product/README.md).
